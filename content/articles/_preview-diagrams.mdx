This article exists only to preview the new `<CycleComparison>` component on PR #6 before merge. It will be removed before final merge so the PR stays scoped to the component itself.

The visual argument: each turn of a development cycle costs the same effort. Same human input, same time budget. Rapid prototyping with agents fits more turns into the same window, so the same effort buys more iterations and the work travels further. Both tracks have the same length (= same time window). The number of wheels along each track is the only thing that changes.

## What to look at

- Does the visual argument land? Same track length, more wheels = more iterations = further?
- Are the wheels readable as wheels (do the spokes help, hurt, or feel like noise)?
- Cypress vs muted: does the contrast carry the "this one is the upside" without being too loud?
- Mobile: do the wheel tracks compress gracefully, or do they get squished/awkward?
- Dark mode: toggle the article header theme. Does it still read?
- Is the caption-per-track useful or noise? Should it be a single overall caption instead?
